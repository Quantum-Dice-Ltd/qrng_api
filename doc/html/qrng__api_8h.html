Detailed Description

QD QRNG API.

Note
All get functions do not zerorise buffer. This means that in cases where the function is unable to read up to the "size" of the passed in buffer "data" the buffer would first be filled with the available data, but the rest of the buffer would retain it's previous value.

QRNG_ERROR_WRONG_DATA_FORMAT 

This is returned if the device mode does not support the function used to generate data.

QRNG_ERROR_INTERNAL_CH_ERROR 

This is returned if there is issue with the communication channel

QRNG_ERROR_NET_RETRIES_EXCEEDED 
QRNG_ERROR_NET_TIMEOUT 
QRNG_ERROR_MULTIPLE_OF_8_REQUIRED 

This is returned if the function requires a length value that is a multiple of 8, for example, 8, 16, 24 etc.

QRNG_ERROR_INSUFFICIENT_ENTHROPY 

This is returned if there is insufficient entropy to generate hashed data with.

QRNG_ERROR_INTERNAL_MEMORY 

Error accessing internal buffer.

This error code is returned if the library couldn't access a buffer for
internal use, because the buffer isn't initialized from the qrng_init() function call. This is avoided by calling the qrng_init function and getting a QRNG_SUCCESS return value.

QRNG_ERROR_READING_DEVICE 

Error while reading from the QRNG device.

This is returned from any of the 'get' operations (i.e. qrng_get, qrng_rand etc.) when the program encounter an error while reading from the device. This can be caused be any of the following:

  1. Incorrect driver configuration or corrupt drivers.
  2. QRNG Hardware disconnection
  3. QRNG Hardware reset
QRNG_ERROR_INCOMPLETE_DATA 

Incomplete data error.

This is returned if any of the 'get' functions couldn't fetch up to the amount of data that was requested.

QRNG_ERROR_OPENING_DEVICE 

Error opening QRNG device.

This is returned from any of the 'get' functions when the device can't be opened to be read from i.e. the QRNG device can be detected on the interface but can't be read from. This is similar to QRNG_ERROR_READING_DEVICE and can be caused by the same conditions.

QRNG_ERROR_NULL_PTR 

Null pointer error.

This is returned if the user passes in a null pointer to the 'qrng_get' function

QRNG_NO_DEVICE_FOUND 

No device found.

If there is no QRNG device connected to the machine or if the drivers are not installed yet.

QRNG_SUCCESS 

Successful operation

int qrng_get_with_ec ( QRNG *  qrng,

s32  eb_size,
float *  cert_val,
s32  cv_size 
)

Get QRNG Random data with Entropy bits and Certification value array

Returns a stream of random numbers as well as the entropy bits and certification value for each group of 64bits(8 bytes). This also implies that users can only read hashed data in multiples of 8 bytes, if the "size" passed in is not equal to 8 the function would return a QRNG_ERROR_MULTIPLE_OF_8_REQUIRED error.

Parameters
[in]QRNG*Pointer to the QRNG object
[out]dataPointer to 8bits array allocation to receive the hashed random numbers
[in]data_sizeSize of allocation "data", must be a multiple of 8
[out]ent_bitsPointer to a 16bits array allocation to receive the Entropy bits
[in]eb_sizeSize of ent_bits array
[out]cert_valPointer to a floats array allocation to receive the certification values
[in]cv_sizeSize of cert_val array
Returns
QRNG_status
